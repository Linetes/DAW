Laboratorio 1: Introducción

Tecnologías

URL: Es la dirección web de alguna página
MIME: Es el protocolo que extiende el formato de e-mail.
Métodos y Códigos de errores HTTP: Protocolo de intercambio de información en World Wide Web.
XML: Lenguaje de documentación, que permite a la máquina y humano leer.
XHTML: Una combinación entre XML y HTML, es más restrictivo.
HTML5: Lenguaje usado para crear contenido en la World Wide Web.
JS: Lenguaje versatil, comunmente usado con HTML5.
php: Lenguaje para servidores y para creación de páginas web, está basado en c.
jsp: Tecnología para crear páginas web dinámicamente con XML, HTML5, etc.
Ajax: Técnicas para la creación de páginas web.
¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es un network de networks y WWW es la forma de accesar a la Internet.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET: Obtener o buscar algo
  • HEAD: Es muy parecido al get sólo que te regresa el título.
  • POST: poner algo.
  • PUT: poner algo también pero ya no es muy usado
  • PATCH: hacerle update a algo
  • DELETE: borrar algo
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

form e input, form es para encerrar e input es para que tenga un get de información.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Código 200 dice que todo está bien.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Depende de la situación, puede ser que algún link lo puso mal o puede ser que lo hizo a propósito para despistar a curiosos.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Depende, ya que es acceso restringido y si un rol necesita acceder a esa página, hay un error en los roles.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Que ya no se soporta su uso, por ejemplo las cosas de diseño en HTML, es mejor ahora poner diseño en el CSS.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 es más nuevo, antes en HTML 4 se ponía todo y era una caos, ahora en HTML5 sólo se pone la estructura, en CSS el diseño y en JS las funciones.
